# Chargable Proxy — 视图文档

> 每个视图从特定角度呈现系统的一个侧面，信息来源于集中模型文件 [chargable-proxy-v1.yaml](../chargable-proxy-v1.yaml) 及补充文档。

## 视图一览

| 视图 | 关注点 | 目标读者 |
|------|--------|----------|
| [业务视图](business.md) | 谁？什么价值？业务流程全景 | 产品、业务方 |
| [领域模型视图](domain-model.md) | 数据结构、状态机、实体关系 | 开发、架构师 |
| [应用架构视图](application-architecture.md) | 子系统组成、交互拓扑、技术栈 | 架构师、开发 |
| [用例视图](use-cases.md) | 各应用的用例、规则、API、关联 | 开发 |
| [页面视图](pages.md) | 前端页面、显示映射、外部链接 | 前端开发、产品 |
| [部署视图](deployment.md) | 物理拓扑、端口、配置、安全 | 运维、开发 |

## 视图关系

```
业务视图                  "为谁提供什么价值？"
  │
  ├──▶ 领域模型视图       "业务涉及哪些核心概念和数据？"
  │       │
  │       ▼
  ├──▶ 应用架构视图       "系统拆分成哪些子系统？怎么交互？"
  │       │
  │       ├──▶ 用例视图   "每个子系统具体做什么？规则是什么？"
  │       │       │
  │       │       └──▶ 页面视图   "用户看到什么界面？"
  │       │
  │       └──▶ 部署视图   "怎么部署？怎么配置？"
```

追溯链路：页面 → 子系统用例 → 系统用例 → 业务用例。页面不直接关联业务用例，而是通过子系统用例 → 系统用例链间接追溯到业务目标。

## 建议阅读顺序

**全局理解**（从业务到技术）：业务 → 应用架构 → 领域模型 → 用例 → 页面 → 部署

**按角色**：
- **产品/业务方**：业务 → 页面
- **架构师**：业务 → 应用架构 → 领域模型
- **后端开发**：应用架构 → 领域模型 → 用例 → 部署
- **前端开发**：应用架构 → 用例 → 页面
- **运维**：应用架构 → 部署
