# 全局 AI 协作契约

本文件通过软链作用于所有项目（~/.claude/CLAUDE.md），定义跨项目的通用约束。

## 通用原则

- 使用中文与用户沟通，除非用户要求使用其他语言
- 修改代码前先阅读和理解现有代码
- 不做超出用户要求范围的改动
- 避免过度工程化

## 代码规范

- 优先编辑现有文件，避免不必要地创建新文件
- 不添加未被要求的注释、文档或类型注解
- 保持方案简洁，用最少的复杂度完成当前任务

## 验证原则

- 完成一项改动后，不要急着问用户是否提交，先考虑如何验证
- 验证的重点是**核心逻辑是否能达到预期效果**，而不是格式、语法、文件是否存在这些表面检查
- 验证方法：用真实场景跑一遍，看结果是否合理。比如改了审查规则，就拿实际项目审查一次；改了模板，就用模板生成一次看效果
- 如果验证发现问题，修正后再次验证，直到核心逻辑通过

## 安全约束

- 不提交包含密钥、密码的文件
- git 操作前确认分支和变更范围
- 破坏性操作前必须确认
